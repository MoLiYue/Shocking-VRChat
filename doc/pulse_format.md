# DG-LAB .pulse 文件格式解析

本文档基于对 DG-LAB 社区分享的 .pulse 波形文件的逆向分析。

## 概述

`.pulse` 文件是 DG-LAB APP 波形编辑器的导出格式，描述了一段完整的电刺激波形。设备播放时需要先转换为 hex op 格式通过 WebSocket 协议发送。

## 整体结构

```
Dungeonlab+pulse:[header=]section0+section+section1+section+section2+section+...
```

- 前缀固定为 `Dungeonlab+pulse:`
- Header 可选，以 `=` 结尾
- 多个 Section 以 `+section+` 分隔
- APP 编辑器固定提供 **10 个 Section 槽位**，未使用的槽位以禁用状态存储

## Header（可选）

格式：`rest_ticks,speed,unknown=`

| 字段 | 含义 | 取值范围 | 示例 |
|------|------|----------|------|
| rest_ticks | 波形结尾的静默时长（tick 数） | 0-99+ | 99 = 9.9s 静默 |
| speed | 播放速度倍率 | 1, 2, 4 | 4 = 四倍速 |
| unknown | 未确认（可能是平衡或其他参数） | - | 8, 16 |

每个 tick = 100ms。

**示例：**
```
99,4,8=    → 结尾静默9.9s，4倍速，第三参数=8
0,1,16=    → 无静默，正常速，第三参数=16
（无header） → 无静默，正常速
```

## Section

格式：`meta/point_list`

### Meta 部分

5 个逗号分隔的整数：`freq_start,freq_end,duration,freq_mode,enabled`

| 位置 | 字段 | 含义 | 取值 |
|------|------|------|------|
| [0] | freq_low | 频率下限（滑块位置）；模式1时为固定频率值 | 0-83 |
| [1] | freq_high | 频率上限（滑块位置）；模式1时忽略 | 0-83 |
| [2] | duration | Section 最小时长（tick 数，每 tick=100ms） | 0-99+ |
| [3] | freq_mode | 频率变化模式 | 1-4 |
| [4] | enabled | 是否启用 | 0=禁用, 1=启用 |

#### duration 与播放规则

Point List 中的每个点代表一个脉冲（1 tick = 100ms = 0.1s）。设 Point List 有 N 个点，duration 为 D：

- 波形单元必须**完整播放**，不能中途截断
- 实际播放次数 = `ceil(D / N)`（至少 1 遍）
- 实际播放时长 = `ceil(D / N) × N × 0.1s`

**示例：**

| 场景 | N (点数) | D (duration) | 循环次数 | 实际时长 |
|------|----------|-------------|---------|---------|
| 夹子用 Section 0 | 17 | 49 | ceil(49/17)=3 | 51 ticks = 5.1s |
| 摸摸拍拍 Section 1 | 8 | 11 | ceil(11/8)=2 | 16 ticks = 1.6s |
| 特别疼 Section 0 | 4 | 8 | ceil(8/4)=2 | 8 ticks = 0.8s |
| 弹球 Section 1 | 8 | 68 | ceil(68/8)=9 | 72 ticks = 7.2s |

> **特殊情况：** 当 N > D 时（波形单元本身就比 duration 长），仍完整播放 1 遍。

#### 频率滑块值

滑块值 0-83 共 84 个档位，对应脉冲间隔（非线性）。间隔越大频率越低、体感越"分明/痛"：

| 滑块范围 | 间隔范围 | 步进 | 档位数 |
|---------|---------|------|--------|
| 0-39 | 10-49ms | 1ms | 40 |
| 40-54 | 50-78ms | 2ms | 15 |
| 55-58 | 80-95ms | 5ms | 4 |
| 59-68 | 100-190ms | 10ms | 10 |
| 69-75 | 200-400ms | 33,33,34 循环 | 7 |
| 76-78 | 450-550ms | 50ms | 3 |
| 79-83 | 600-1000ms | 100ms | 5 |

> 200-400ms 区间步进为 33,33,34 循环（每 3 步凑 100ms），产生：200, 233, 266, 300, 333, 366, 400。
> 400ms 已包含在该区间末尾，下一区间从 450ms 开始。

关键参考值：

| slider | 间隔 | 频率 | 体感 |
|--------|------|------|------|
| 0 | 10ms | 100 Hz | 最高频，柔和/震动感 |
| 20 | 30ms | 33 Hz | 中高频 |
| 40 | 50ms | 20 Hz | 中频 |
| 54 | 78ms | 12.8 Hz | 中低频 |
| 59 | 100ms | 10 Hz | 低频，开始有明显脉冲感 |
| 69 | 200ms | 5 Hz | 很低频，每下都很明显 |
| 75 | 400ms | 2.5 Hz | 极低频 |
| 83 | 1000ms | 1 Hz | 最低频，1秒一次 |

> .pulse 中的 freq_start/freq_end 直接就是 0-83 的滑块档位值。

#### 频率模式

| 值 | 模式 | 描述 |
|----|------|------|
| 1 | 固定 | 整个 Section 使用 freq_low 作为固定频率，freq_high 忽略 |
| 2 | 线性渐变 | 频率从 freq_low 到 freq_high 线性过渡（跨整个 Section 时长） |
| 3 | 循环渐变 | 每个波形单元内频率从 freq_low 到 freq_high 循环一次 |
| 4 | 分段递进 | 每重复一次波形单元，频率递进一级（从 freq_low 到 freq_high） |

> 注意：freq_low 不一定小于 freq_high。当 freq_low > freq_high 时，频率变化方向相反（从低频向高频渐变）。

#### enabled 字段

| 值 | 含义 |
|----|------|
| 1 | Section 启用，参与波形播放 |
| 0 | Section 禁用（空槽位或被用户关闭），**不应参与播放** |

> **注意：** APP 编辑器始终导出 10 个 Section 槽位。未使用的槽位通常为：
> `0,20,20,1,0/0-1,20-1`（禁用状态的默认占位符）

### Point List 部分

格式：`value-flag,value-flag,...`

| 字段 | 含义 |
|------|------|
| value | 强度值，范围 0-100（百分比） |
| flag | 点类型：1=锚点（用户手动设置），0=自动插值点 |

Point List 描述了 **一个波形单元** 的强度曲线形状。每个点对应一个脉冲（1 tick = 0.1s）。该单元按照 duration 的规则循环播放（见上文"duration 与播放规则"）。

#### 锚点 vs 自动点

- **锚点 (flag=1)**：用户在编辑器中手动放置的关键帧
- **自动点 (flag=0)**：编辑器自动在锚点之间生成的插值点

在转换时，通常对所有点（无论 flag）进行线性重采样到目标时长。flag 的实际作用是编辑器内的 UI 交互行为（拖动锚点时自动点跟随变化）。

## 完整示例解析

以 `pulse-摸摸拍拍-9049275.pulse` 为例：

```
Dungeonlab+pulse:
  0,0,25,1,1/0.00-1,16.67-0,...,100.00-1,...,45.08-1    ← Section 0 (启用)
  +section+
  0,54,11,1,1/100.00-1,...,0.00-1                        ← Section 1 (启用)
  +section+
  1,20,13,1,1/100.00-1,0.00-1                           ← Section 2 (启用)
  +section+
  0,20,20,1,0/0-1,20-1                                  ← Section 3 (禁用/占位)
  +section+
  ...（重复5次相同的禁用占位）
```

**Section 0** `0,0,25,1,1`：
- 频率固定 240Hz（最柔和）
- 时长 2.5s
- 启用
- 30 个点描述"摸上去再松手"的曲线

**Section 1** `0,54,11,1,1`：
- 频率固定 116Hz（中等偏柔）
- 时长 1.1s
- 启用
- 从 100% 线性下降到 0%

**Section 2** `1,20,13,1,1`：
- 频率固定 238Hz（柔和）
- 时长 1.3s
- 启用
- 从 100% 快速降到 0%

**Section 3-9** `0,20,20,1,0`：
- 禁用（enabled=0）
- 占位符，不参与播放

**实际播放时长** = 25 + 11 + 13 = 49 ticks = 4.9s（仅启用的 Section）

## 转换为设备协议

转换后生成 hex op 序列，每个 op 为 16 位十六进制字符串：

```
FFFFFFFF SSSSSSSS
│        │
│        └─ 4 字节强度 (每字节 0x00-0x64 = 0-100%)
└────────── 4 字节频率 (每字节 0x0A-0xF0 = 10-240)
```

每个 op 代表 100ms 播放时间（4 个 25ms 子步骤）。

**示例：**
```
F0F0F0F000050A0F
│       │
│       └─ 强度: 0%, 5%, 10%, 15% (4个25ms子步骤逐渐增强)
└───────── 频率: 240,240,240,240 (固定高频/柔和)
```

## 已知问题

1. **转换器 bug — enabled 字段误解**：当前 `dglab_pulse_converter.py` 将 `meta[4]=0` 理解为"强度值类型=绝对值0-20"，实际应为"禁用"。这导致转换结果中包含不该播放的占位 Section 数据。

2. **转换器 bug — duration/循环逻辑**：当前转换器将 Point List 线性重采样到 `duration × 4` 个子步骤，而非按"完整循环播放"规则处理。正确做法应该是循环 `ceil(duration / N)` 遍波形单元，每个点对应 1 个 op（4 个子步骤）。

3. **插值算法**：转换器使用线性插值连接关键帧点。DG-LAB APP 内部可能使用贝塞尔曲线或其他平滑算法，可能造成曲线平滑度差异（实际体感差异极小）。

4. **Header 第三参数**：含义未确认。观察到的值包括 1, 8, 16 等，可能与频率、通道平衡或其他全局设置相关。
